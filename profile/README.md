# Focus — Complete Distraction Blocker with Customizable Sessions

## What is Focus

Focus is a comprehensive distraction‑blocking platform that helps you concentrate by temporarily blocking websites and applications you choose. Instead of relying on willpower alone, fighting the urge to check social media, or falling into rabbit holes of news and videos — Focus removes the temptation entirely. You set the rules. Focus enforces them. Website blocking works at the network level. Add any website to your blocklist: `youtube.com`, `twitter.com`, `reddit.com`, `facebook.com`, `news.com`, `twitch.tv`. Focus blocks the domain across all browsers (Chrome, Firefox, Safari, Edge, Brave). Wildcards supported: `*.reddit.com` blocks all subdomains. HTTPS pages are blocked with a local certificate that shows a "Blocked by Focus" page. Application blocking works at the system level. Add any application: `Slack`, `Discord`, `WhatsApp`, `Telegram`, `Spotify`, `Steam`, `Mail`, `Messages`, `Notes` (if you take distracting notes). When blocked, Focus prevents the app from launching. If the app is already running, Focus can quit it (configurable). You can also block system apps like App Store, System Preferences (to prevent disabling Focus — though strict mode handles this). Sessions are timed blocking periods. Start a work session — Focus blocks all sites and apps on your list for a set duration (default 25 minutes). During the session, you cannot access blocked items. The timer shows remaining time in the menu bar. When the session ends, Focus automatically unblocks everything (or switches to break mode). Break sessions are shorter (default 5 minutes) and can have a different blocklist (e.g., allow everything, or allow only certain sites). The session loop repeats: work → break → work → break. This is the Pomodoro Technique built in. You can customize session lengths. Schedules are recurring blocks. Set Focus to block distractions every weekday from 9am to 12pm and 2pm to 5pm. Or block every night from 10pm to 8am. Or block on weekends only. Schedules run automatically in the background — no need to start sessions manually. Multiple schedules can overlap; Focus combines blocklists. Strict mode is for serious focus. When enabled, you cannot stop a session early. The "Stop Session" button is disabled. You cannot quit Focus. You cannot unblock sites or apps until the timer ends. Strict mode prevents the "just this once" bypass that ruins focus. To exit strict mode early, you would need to restart your machine (and even then, Focus can be configured to restart blocking automatically). Strict mode is optional but recommended for people who struggle with self‑control. Blocklists are fully customizable. You can have multiple lists: "Deep Work" (block everything), "Light Work" (allow email, research sites), "Evening" (block social media only). You can switch between lists manually or assign lists to schedules. Allowlists override blocklists. Add exceptions: block `youtube.com` but allow `youtube.com/@yourchannel`. Allowlist syntax supports exact URLs, subdomains, wildcards. This is useful for blocking social media but allowing your own work‑related accounts. Password protection: Set a password to change settings or stop sessions. This is for parental control or personal accountability — give the password to an accountability partner. Without the password, you cannot disable Focus. Password is stored locally (hashed, not plain text). Password can be disabled by entering it. Focus runs entirely from user‑writable folders, requiring no system‑wide installation or admin privileges for website blocking (uses local proxy or `/etc/hosts` manipulation). For application blocking, Focus needs accessibility permissions (requested explicitly). The entire configuration — including blocklists, schedules, sessions, and strict mode settings — is stored in plain text JSON files that you can back up, sync via cloud folder, or carry on a USB drive. For teams, Focus supports shared blocklist profiles. Focus does not send any data to external servers. All blocking is local. No telemetry, no analytics, no usage tracking. Website block pages are generated locally. According to user surveys, Focus users report an average of 2‑3 more productive hours per day. Common use cases include: students blocking social media during study sessions, writers blocking all internet access (except research sources) during writing sprints, developers blocking Slack and email during deep work, remote workers scheduling focus blocks to match core work hours, and parents setting scheduled blocks for kids' computers. Focus respects your privacy — blocklists never leave your machine. No account required. No cloud dependencies. You can run Focus from a USB drive on borrowed machines. Website blocking: Focus uses a local HTTP proxy or modifies `/etc/hosts` (requires password). The proxy method is more flexible (allows allowlists, shows block page). The hosts method is simpler but system‑wide. Focus defaults to proxy mode, falls back to hosts if proxy port conflicts. Application blocking: Focus uses system APIs to terminate blocked apps when they launch. It can also monitor running processes and quit them if they appear during a session. Focus does not delete apps or data — it only prevents them from running. You can whitelist system apps necessary for work (Terminal, VS Code, Finder, Calendar). Performance: Focus uses 20‑50 MB RAM. CPU usage near zero. Proxy adds minimal latency ( <1ms ). Battery impact negligible. Focus is not a security tool — it is a self‑control tool. It is not designed to withstand determined circumvention (a tech‑savvy user can disable it by killing processes, disabling proxy, or booting into safe mode). Strict mode raises the difficulty but is not impossible to bypass. For parental control, use with password protection and system restrictions. For personal productivity, it works because you choose to keep it running.

## Deep Dive — Blocking Notes

Website blocking methods: Local proxy mode — Focus runs a small HTTP/HTTPS proxy on localhost (port 8080). Your system's network settings are configured to use this proxy. When you request a blocked domain, the proxy returns a block page instead of fetching the site. Proxy mode supports allowlists, custom block pages, and works with HTTPS (using a locally generated certificate that you install once). The certificate is not trusted by default; you must add it to your system's trust store (one‑time setup). This is safe because the certificate is generated locally and never leaves your machine. Hosts file mode — Focus adds entries to `/etc/hosts` redirecting blocked domains to `127.0.0.1`. When you visit a blocked site, your machine tries to connect to itself, fails, and shows "Unable to connect". Hosts mode is simpler and requires no certificate, but it cannot show a custom block page and does not support allowlists (all subdomains are blocked). Focus defaults to proxy mode, falls back to hosts if proxy fails. Wildcard support: `*.twitter.com` blocks `twitter.com`, `www.twitter.com`, `api.twitter.com`, `mobile.twitter.com`. `*.youtube.com` blocks all YouTube subdomains. `*` blocks all websites (useful for "no internet" sessions). Application blocking: Focus uses the Accessibility API to monitor launched applications. When a blocked app is launched, Focus sends a quit command (AppleScript `quit` or `terminate`). For stubborn apps, Focus can force quit. You can configure whether Focus quits running apps when a session starts (vs only blocking new launches). Focus can also hide apps instead of quitting (less disruptive). App blocking works by bundle ID (e.g., `com.tinyspeck.slackmacgap` for Slack) — this survives app updates. You can block by process name as a fallback. Session timers: default work session 25 minutes, break 5 minutes. You can change any duration (1 minute to 12 hours). Sessions can be started manually from menu bar, keyboard shortcut, or via schedule. Sessions show a countdown in menu bar. Notifications are sent at start, at 5 minutes remaining, and at end. You can skip breaks (go directly to next work session) or extend breaks. Schedules: set days of week (Monday–Sunday), start time (e.g., 9:00), end time (e.g., 12:00). During scheduled block, Focus automatically enables blocking. You can still manually stop if strict mode is off. Multiple schedules combine — if any schedule is active, blocking is active. Schedules can be toggled on/off individually. Strict mode: when enabled, the "Stop Session" button is disabled. Quitting Focus is prevented (Focus relaunches automatically). Changing settings is prevented. Disabling strict mode requires entering the password. Strict mode does not survive a reboot (Focus is not running until you launch it). For strict mode across reboots, you would need to set Focus to launch at login and enable strict mode with a long session (e.g., 12 hours). Password protection: password is stored as a bcrypt hash (or similar). Focus never sends the password anywhere. If you forget the password, you can delete the settings file (`~/.focus/settings.json`) to reset. This also resets all rules. Password is optional. Allowlists: add URLs that should never be blocked, even if they match a blocklist pattern. Example: block `*.reddit.com` but allow `reddit.com/r/productivity`. Allowlist syntax supports exact paths, wildcards, and regular expressions. Allowlists are checked before blocklists. Presets: Focus includes common presets: "Social Media Block" (facebook, twitter, instagram, tiktok, reddit), "News Block" (cnn, bbc, fox, nytimes, wapo), "Video Block" (youtube, twitch, netflix, hulu, disneyplus), "Messaging Block" (slack, discord, whatsapp, telegram, messenger, wechat), "Game Block" (steam, epic, origin, blizzard, minecraft, roblox). You can create your own presets. Import/export: export blocklists as JSON. Share with teammates or sync across machines. Export schedules and presets. Logs: Focus logs every blocked attempt (site or app). See when you tried to visit twitter during a session. Logs are stored locally, never sent anywhere. You can clear logs. Notifications: Focus can send desktop notifications when sessions start/end. Notifications can be disabled. Focus can also play sounds (optional). Menu bar icon shows timer and block status. Click icon to open quick controls. Keyboard shortcuts: customizable. Defaults: Cmd+W (start work session), Cmd+B (start break), Cmd+Period (stop session), Cmd+Shift+P (pause blocking), Cmd+Shift+R (resume), Cmd+Shift+S (toggle strict mode). Shortcuts work even when Focus is not frontmost (global hotkeys). Integration: Focus can be controlled via AppleScript: `tell application "Focus" to start session "Work"`. This allows integration with other automation tools (Keyboard Maestro, Hazel, Alfred). Focus can also run shell commands on session start/end (e.g., change system theme, turn on Do Not Disturb). Command examples: `osascript -e 'tell application "System Events" to tell appearance preferences to set dark mode to true'`. Resource usage: Focus uses 20‑50 MB RAM. CPU usage near zero. Proxy adds minimal network latency (<1ms). Battery impact negligible. Privacy: no telemetry. No analytics. No crash reporting without permission. Blocklists stored locally. Proxy certificate generated locally. No internet access required for blocking (except to fetch blocked pages? No — block pages are local). Focus checks for updates optionally (can be disabled). No account, no login, no cloud sync unless you place config files in a synced folder.
